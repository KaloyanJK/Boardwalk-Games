# Project Planning - User Stories

## User Story 1: User-Friendly Navigation and Responsive Design (Must Have)

**User Story:**\
As a first-time visitor, I need easy navigation and a user-friendly
design, including a responsive layout for my device, so I can find
information quickly and efficiently without frustration.

**Acceptance Criteria:** - The website is fully responsive across
various devices and screen sizes. - The site layout and navigation are
intuitive, allowing easy access to different sections.

**Tasks:** - Apply responsive design principles using Bootstrap. -
Arrange layout and navigation using best practices.

------------------------------------------------------------------------

## User Story 2: High-Quality Images and Engaging Descriptions (Must Have)

**User Story:**\
As a casual gamer, I want to see high-quality images and engaging
descriptions of the café's ambience and available games.

**Acceptance Criteria:** - Homepage includes a carousel with
high-quality images. - Descriptions are clear and concise. - Layout is
uncluttered.

**Tasks:** - Add carousel with images. - Add descriptions. - Design
homepage layout.

------------------------------------------------------------------------

## User Story 3: Location, Contact Details, and Opening Hours (Must Have)

**User Story:**\
As a prospective customer, I need to find location, contact details, and
opening hours easily.

**Acceptance Criteria:** - Dedicated section exists. - Section is
accessible from all pages.

**Tasks:** - Create section with provided info. - Ensure visibility
across site.

------------------------------------------------------------------------

## User Story 4: Booking Inquiry Form (Must Have)

**User Story:**\
As a customer, I want to book a table or join an event.

**Acceptance Criteria:** - Form includes: Name, Email, Phone, Event
Type, Message. - All fields required. - Redirect to success page on
submit.

**Tasks:** - Build form. - Add validation. - Create success page.

------------------------------------------------------------------------

## User Story 5: Pricing and Student Discount (Should Have)

**User Story:**\
As a student gamer, I want to see pricing and discounts.

**Acceptance Criteria:** - Pricing is visible. - Discounts are
highlighted.

**Tasks:** - Display pricing. - Highlight discounts.

------------------------------------------------------------------------

## User Story 6: Testimonials (Could Have)

**User Story:**\
As a prospective customer, I want to read reviews.

**Acceptance Criteria:** - Testimonials section exists. - Easy to read.

**Tasks:** - Create section. - Add testimonials.

------------------------------------------------------------------------

## User Story 7: Newsletter Sign-Up (Could Have)

**User Story:**\
As a regular customer, I want updates.

**Acceptance Criteria:** - Form in footer. - Available on all pages.

**Tasks:** - Add form to footer.

------------------------------------------------------------------------

## User Story 8: Game Library

**User Story:**\
As a customer, I want to view available games.

**Acceptance Criteria:** - 8 games total. - 4 new, 4 favourites. - Each
has name, image, description. - Responsive layout.

**Tasks:** - Design layout. - Build page. - Add content. - Test
responsiveness.

------------------------------------------------------------------------

## User Story 9: Game Reservation

**User Story:**\
As a customer, I want to reserve a game.

**Acceptance Criteria:** - Includes game, date, name, contact. -
Validates input. - No past dates. - Shows confirmation.

**Tasks:** - Build form. - Add validation. - Test usability.
